# SmartBus ML — Data Schema and Preprocessing Guide

---

## Overview

The SmartBus ML pipeline uses a **synthetic dataset** that mirrors the Phase III
PostgreSQL schema. Data is generated by `generate_data.py` and preprocessed into
ML-ready feature matrices by `preprocessing.py`.

---

## Raw Data Files (`data/raw/`)

### `users.csv`

| Column | Type | Description |
|--------|------|-------------|
| `user_id` | int | Unique user identifier (1–300) |
| `email` | string | Unique email address (`firstname.lastnamei@example.com`) |
| `full_name` | string | First and last name |
| `home_stop` | string | User's primary boarding stop |
| `travel_profile` | enum | `commuter` / `frequent` / `occasional` / `rare` |
| `registration_date` | date | `YYYY-MM-DD` — when the user registered |

**home_stop values:** Downtown Terminal, Airport Station, University, Bus Depot, City Center, Harbor

**travel_profile distribution:** occasional (28.7%), frequent (27.7%), commuter (24.0%), rare (19.7%)

---

### `routes.csv`

| Column | Type | Description |
|--------|------|-------------|
| `route_code` | string | Unique route identifier (e.g., `SB-101`) |
| `from_stop` | string | Departure stop name |
| `to_stop` | string | Arrival stop name |
| `departure_time` | time | `HH:MM` format |
| `arrival_time` | time | `HH:MM` format |
| `unit_price` | float | Fare per passenger in USD |
| `seats_total` | int | Total seat capacity |
| `seats_available` | int | Current available seats (starts equal to seats_total) |

**10 routes** connecting 6 stops. Price range: $7.00–$18.00.

---

### `bookings.csv`

| Column | Type | Description |
|--------|------|-------------|
| `booking_id` | int | Auto-incremented primary key |
| `booking_reference` | string | Human-readable reference (`BK-000001`) |
| `user_id` | int | Foreign key → users.user_id |
| `customer_email` | string | Denormalized email (matches user) |
| `route_code` | string | Foreign key → routes.route_code |
| `from_stop` | string | Denormalized departure stop |
| `to_stop` | string | Denormalized arrival stop |
| `trip_date` | date | `YYYY-MM-DD` — scheduled trip date |
| `trip_type` | string | Always `one-way` in this dataset |
| `passengers` | int | Number of passengers (1–4) |
| `total_amount` | float | `unit_price × passengers` |
| `current_state` | enum | `COMPLETED` or `CANCELLED` |
| `created_at` | date | Same as trip_date in synthetic data |

**5,174 total bookings:** 3,858 COMPLETED (74.6%), 1,316 CANCELLED (25.4%)

## Processed Data Files (`data/processed/`)

### `interactions.csv`

One row per **(user_id, route_code)** pair that has at least one COMPLETED booking.

| Column | Type | Description |
|--------|------|-------------|
| `user_id` | int | References users.user_id |
| `route_code` | string | References routes.route_code |
| `booking_count` | int | Number of COMPLETED bookings for this pair |
| `implicit_rating` | float | MinMax-scaled booking_count (0.0–1.0 across all pairs) |

**1,507 pairs** across 288 users and 10 routes. Matrix density: 50.2%.

`implicit_rating` is the **training target** for the recommendation model.
Higher values indicate stronger preference.

---

### `route_features.csv`

One row per route with engineered features for content-based filtering (22+ columns).

| Column group | Columns | Description |
|-------------|---------|-------------|
| Identity | `route_code`, `from_stop`, `to_stop` | Route identifiers |
| Numeric | `unit_price`, `price_scaled` | Raw price and MinMax-scaled price |
| Categorical | `price_tier`, `departure_time`, `departure_bucket` | Tier and hour bucket |
| Capacity | `seats_total` | Total seat capacity |
| One-hot origin | `origin_Downtown_Terminal`, …, `origin_Harbor` | 6 columns |
| One-hot destination | `dest_Downtown_Terminal`, …, `dest_Harbor` | 6 columns |
| One-hot hour | `hour_early_morning`, …, `hour_evening` | 5 columns |
| One-hot tier | `tier_LOW`, `tier_MEDIUM`, `tier_HIGH` | 3 columns |

---

### `user_profiles.csv`

One row per user with aggregated booking behaviour.

| Column | Type | Description |
|--------|------|-------------|
| `user_id` | int | References users.user_id |
| `email` | string | User email |
| `home_stop` | string | Declared home stop |
| `travel_profile` | enum | From raw users data |
| `total_bookings` | int | Count of COMPLETED bookings |
| `frequent_origin` | string | Most common departure stop in bookings |
| `avg_spend` | float | Mean total_amount across completed bookings |
| `preferred_tier` | enum | Most booked price tier (LOW/MEDIUM/HIGH) |
| `preferred_hour` | enum | Most booked departure hour bucket |

---

### `train.csv` and `test.csv`

Same schema as `interactions.csv`. Per-user temporal hold-out split:
- **Train:** 1,125 pairs (74.7%)
- **Test:** 382 pairs (25.3%)

Users with only 1 interaction remain in train only (no test row).

## How to Regenerate the Dataset

```sh
# From project root
source ml/venv/bin/activate

# 1. Generate raw data (idempotent — safe to re-run)
python ml/generate_data.py

# 2. Preprocess into ML-ready files
python ml/preprocessing.py
```

Both scripts are **deterministic** (seed=42). Re-running produces identical output.

---

## Data Quality Guarantees

The preprocessing pipeline asserts:
- No orphan bookings (every user_id and route_code must exist in reference tables)
- No duplicate booking_reference values
- No missing customer_email

If any assertion fails, the script exits with a descriptive error before writing any output.

## Mapping to Phase III PostgreSQL Schema

| Raw CSV Column | PostgreSQL Table | Column |
|----------------|-----------------|--------|
| `booking_reference` | `booking_process_instances` | `booking_reference` |
| `customer_email` | `booking_process_instances` | `customer_email` |
| `route_code` | `booking_process_instances` / `schedule_routes` | `route_code` |
| `from_stop` | `booking_process_instances` | `from_stop` |
| `to_stop` | `booking_process_instances` | `to_stop` |
| `total_amount` | `booking_process_instances` | `total_amount` |
| `current_state` | `booking_process_instances` | `current_state` |
| `unit_price` | `schedule_routes` | `unit_price` |
| `seats_available` | `schedule_routes` | `seats_available` |

The synthetic dataset can be replaced by a live PostgreSQL export once the system
accumulates real bookings. Only `generate_data.py` needs to be swapped for a DB query;
`preprocessing.py` is schema-agnostic.
